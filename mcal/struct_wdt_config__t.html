Detailed Description

Wdt configuration structure.

Example to configure WDT:

WdtConfig_t config = {0};
config.timeout.unit = enTimeUnit_ms; // in milliseconds
config.timeout.value = 5000; // 5000 ms
config.hardwareCfg = PCF_NULL; // No hardware specific config

◆ timeout

TimeInterval_t WdtConfig_t::timeout

The time after which WDT will reset the system. Incase user passes a timeout value that is not supported, the driver will select the appropriate timeout value. User can query the actual timeout value set by calling pcf_Wdt_getTimeout(TimeInterval_t *timeout) User will have to call pcf_Wdt_refresh() before the timeout occurs to prevent reset.

Hardware specific configuration structure. Refer to native specific header file (if they exists) under pcf/peripherals/gpio/<native specific folder> for details. Set to PCF_NULL if native specific file does not exist
