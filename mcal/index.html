Introduction

The @pcf/mcal is a header only library that provides API definitions for microcontroller (MCU) and peripheral operations. It serves as an abstraction layer, simplifying the interaction with hardware components.

The mcal header encapsulates low-level functionalities of the MCU and peripheral devices, offering a standardized interface for accessing and controlling hardware resources.

By utilizing the mcal header, developers can write portable code that is independent of the specific MCU architecture or peripheral implementation, promoting code reusability and portability.

The mcal library defines a set of APIs that enable tasks such as GPIO manipulation, interrupt handling, timer and PWM control, communication protocols (UART, SPI, I2C), analog-to-digital conversion, and more.

With the mcal header, developers can focus on writing application-level logic without needing to delve into the intricate details of MCU registers and peripheral configurations, resulting in faster development cycles and increased productivity.

@pcf/mcal library package provides low-level functionality for the following:

  • pcf/mcu/
    • mcu (Clock and cpu info)
    • efmi (Embedded flash memory interface)
    • interrupt controller
  • pcf/peripherals/
    • gpio (General purpose input output)
    • gpio_bank (Bank of gpio)
    • adc (Analog to digital convertor)
    • dac (Digital to analog convertor)
    • periodic_timer (Periodic timer)
    • rtc (Real time clock)
    • PWM (Pulse with modulation signal)
    • ext_int (External interrupt)
    • qei (Quadrature Encoder Interface)
    • wdt (Watch dog timer)
    • input_capture (Capture timestamp value for a receive input signal)
    • output_compare (Trigger an output based on a timestamp)
    • uart (Universal asynchronous receiver-transmitter)
    • spi (Serial peripheral interface)
    • i2c (Inter-Integrated circuit)
    • can (Controller area network)

How to use

To use the package item include the pcf/<package_module_name>/<package_item_name>/<package_item_name>.h in your application.
Eg.: #include <pcf/peripherals/gpio/gpio.h>

Required compile macros:

Following macros are required for building for any mcu with no operating system:
PCF_CONF_ARCH_DEVICE_VENDOR - Valid values stm, nxp or infineon
PCF_CONF_ARCH_DEVICE_SERIES - Valid values stm32f3, stm32f4, lpc1700 or xmc4000
PCF_CONF_ARCH_DEVICE - Valid values stm32f302r8, stm32f405rg, lpc1768 or xmc4700

Note
Macros which start with "__" or "_" and doesn't end with "__" are reserved to support internal infrastructure and not meant to be used in your code.
