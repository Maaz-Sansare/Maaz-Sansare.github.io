Detailed Description

SPI refers to Serial Peripheral Interface is a synchronous serial communication protocol used to transfer data between microcontrollers and various peripheral devices like sensors, displays, and memory chips.

SPI offers a high-speed, full-duplex data exchange with a master-slave architecture, making it widely employed in embedded systems for efficient data transfer.

Note
Input validation is done only when package is built in debug mode.

Sets the Tx and Rx interrupt priority of a particular Spi_t object depending on mode of spi's Tx and Rx.

Note
spi Tx can be either in DMA,INT or POL mode.
spi Rx can be either in DMA,INT or POL mode.
The mode of spi is decided via config at time of creation of Spi_t object using pcf_Spi_new.
Parameters
spi[IN] - A pointer to Spi_t object.
tx_priority[IN] - NVIC priority level for spi Tx, lower value indicates higher priority
tx_sub_priority[IN] - NVIC sub-priority level for spi Tx, lower value indicates higher sub-priority
rx_priority[IN] - NVIC priority level for spi Rx, lower value indicates higher priority
rx_sub_priority[IN] - NVIC sub-priority level for spi Rx, lower value indicates higher sub-priority
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Spi_t object is passed.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid Spi_t object is passed.(Checked only in debug mode)
Note
If uart Tx and Rx both are in INT mode, then only rx_priority and rx_sub_priority are taken into consideration while setting the priority.

◆ pcf_Spi_read()

PCF_MCAL_API pcf_status_t pcf_Spi_read ( Spi_t *const  spi,
Buffer_t *const  buffer,
pcf_size_t *const  actual_size,
const uint32_t  timeout_ms 
)

Receive/Read data on spi.

Run in polling (blocking) mode if rxBufferSize is set to 0 or if the system could not allocate memory for the RxBuffer during creation Run in DMA mode (Default) or interrupt mode (if DMA is not available or busy) if rxBufferSize is set to any nonzero positive value.

The user provides a pointer to Buffer_t which stores the buffer address and size to be read in bytes. The function does not guarantee reading the size specified by the user. The actual number of bytes read is returned in actual_size .

Note
In Polling/Blocking mode, This function blocks until all bytes specified by buffer are received or timeout is met.
In DMA or interrupt/Non-blocking mode, this function does not block for the actual receive on the wire. It does not guarantee the read/receive of all bytes specified by buffer . It depends on the free space available in the internal buffer. The actual number of bytes received is saved in actual_size .
User can call pcf_Spi_getRxBufferCount function to check how much data is present in the receive buffer.
Parameters
spi[IN] - A pointer to Spi_t object.
buffer[IN/OUT] - Pointer to Buffer_t, stores the data into the buffer of specified length.
actual_size[OUT] - Pointer to pcf_size_t, stores actual number of bytes read.
(Optional parameter, pass PCF_NULL if not required.)
timeout_ms[IN] - Timeout in ms, will be ignored in non-blocking mode.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Spi_t object is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid Spi_t object is passed.(Checked only in debug mode)
  • PCF_S_TIMEOUT : Operation Timed out.
Note
If value of databits is set to greater than 8 bits. Then, 2 consecutive bytes will represent 1 single frame. In this case value of size should be in a multiple of 2. If the value of size is an odd value, then a 0 is padded in the last byte of the data.

Send/Write data over spi bus.

Run in polling (blocking) mode if txBufferSize is set to 0 or if the system could not allocate memory for the buffer. Run in DMA mode (Default) or interrupt mode (If DMA is not available or is busy) if txBufferSize is set to any nonzero positive value.

Note
In Polling/Blocking mode, This function blocks until all bytes specified by buffer are sent or timeout is met.
In non-blocking mode, this function does not block for the actual transmission on the wire. It does not guarantee the transmission of all bytes specified by the user (size). It depends on the free space available in the internal buffer. The actual number of bytes sent is saved in the actual_size variable.
Parameters
spi[IN] - A pointer to Spi_t object.
buffer[IN] - Pointer to Buffer_t, writes the data in buffer of specified length to the spi.
actual[OUT] - Pointer to pcf_size_t, stores actual number of bytes sent successfully. (Optional parameter, pass PCF_NULL if not required.)
timeout_ms[IN] - Timeout in ms, will be ignored in non-blocking mode.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Spi_t object is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid Spi_t object is passed.(Checked only in debug mode)
  • PCF_S_BUSY : Spi is busy when a previous transmission is already in progress.
  • PCF_S_TIMEOUT : Operation Timed out.
Note
If value of databits is set to greater than 8 bits. Then, 2 consecutive bytes will represent 1 single frame. In this case value of size should be in a multiple of 2. If the value of size is an odd value, then a 0 is padded in the last byte of the data.
