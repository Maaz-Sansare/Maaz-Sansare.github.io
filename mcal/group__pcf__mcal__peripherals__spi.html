◆ pcf_Spi_setPriority()

PCF_MCAL_API pcf_status_t pcf_Spi_setPriority ( const Spi_t *const  spi,
const uint32_t  tx_priority,
const uint32_t  tx_sub_priority,
const uint32_t  rx_priority,
const uint32_t  rx_sub_priority 
)

Set the Tx and Rx interrupt priority of a particular Spi_t object depending on mode of spi's Tx and Rx.

Note
spi Tx can be either in DMA,INT or POL mode.
spi Rx can be either in DMA,INT or POL mode.
The mode of spi is decided via config at time of creation of Spi_t object using pcf_Spi_new.
Parameters
spi[IN] - Pointer to Spi_t.
tx_priority[IN] - NVIC priority level for spi Tx, lower value indicates higher priority
tx_sub_priority[IN] - NVIC sub-priority level for spi Tx, lower value indicates higher sub-priority
rx_priority[IN] - NVIC priority level for spi Rx, lower value indicates higher priority
rx_sub_priority[IN] - NVIC sub-priority level for spi Rx, lower value indicates higher sub-priority
Returns
pcf_status_t Valid values :
  • PCF_SPI_S_SUCCESS : Successful operation.
  • PCF_SPI_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_SPI_S_INVALID_ARG : Invalid Spi_t object was passed.(Checked only in debug mode)
Note
If spi Tx and Rx both are in INT mode, then only "rx_priority" and "rx_sub_priority"

are taken into consideration while setting the priority.

Register a spi slave.

Call this function in the begining before doing any communication with slave. The function will give an id which can be used to select and unselect the slave.

Parameters
spi[IN] - Pointer to Spi_t.
slave_select_pin[IN] - PortPin information to select the slave
slave_id[OUT] - Pointer to store slave id
Returns
pcf_status_t Valid values :
  • PCF_SPI_S_SUCCESS : Successful operation.
  • PCF_SPI_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_SPI_S_INVALID_ARG : Invalid Spi_t object was passed.(Checked only in debug mode)
  • PCF_SPI_S_INVALID_OPERATION : Spi is not in master mode.
  • PCF_SPI_S_ALREADY_IN_USE : Slave port pin is already in use.
  • PCF_SPI_S_INTERNAL_ERROR : Max slave count reached or slave registeration failed due to some internal error.

)

Read data on spi bus.

If the system could not allocate memory for the RxBuffer during creation then the function is blocking else it is non-blocking. The user provides a pointer to a data buffer and size (the number of bytes) that need to be read. The function does not guarantee reading the size specified by the user. The actual number of bytes read is returned to user in variable actual_size.

User may Call pcf_Spi_getRxBufferCount() function to check if data is present in receive buffer or not. Call this function to read the buffer.

Parameters
spi[IN] - Pointer to Spi_t.
data[OUT] - Pointer to data buffer
size[IN] - Total number of bytes to be read
actual_size[OUT] - Actual number of bytes read
Returns
pcf_status_t Valid values :
  • PCF_SPI_S_SUCCESS : Successful operation.
  • PCF_SPI_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_SPI_S_INVALID_ARG : Invalid Spi_t object was passed.(Checked only in debug mode)
Note
If value of databits is set to greater than 8 bits. Then, 2 consecutive bytes will represent 1 single frame. In this case value of size should be in a multiple of 2. If the value of size is an odd value, then a 0 is padded in the last byte of the data.

◆ pcf_Spi_write()

PCF_MCAL_API pcf_status_t pcf_Spi_write ( const Spi_t *const  spi,
const pcf_byte_t *  data,
const pcf_size_t  size,
pcf_size_t *  actual_size 
)

Send data on spi bus.

Run in polling (Blocking) mode if txBufferSize is set to 0. Run in DMA or interrupt (Non-Blocking) mode if txBufferSize is set to any nonzero positive value.

In Polling/Blocking mode This function blocks until all bytes specified by user (size) are sent on the wire. The function will guarantee transmission of all bytes.

In Non-Blocking mode This function will not block for actual transmission on the wire. The function will not guarantee transmission of all bytes specified by user (size). It depends on free space available in internal buffer. The actual number of bytes sent is saved in variable actual_size.

Parameters
spi[IN] - Pointer to Spi_t.
data[IN] - Pointer to data buffer
size[IN] - Total number of bytes to be sent
acutal_size[OUT] - The actual number of bytes sent
Returns

  • PCF_SPI_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_SPI_S_INVALID_ARG : Invalid Spi_t object was passed.(Checked only in debug mode)
Note
If value of databits is set to greater than 8 bits. Then, 2 consecutive bytes will represent 1 single frame. In this case value of size should be in a multiple of 2. If the value of size is an odd value, then a 0 is padded in the last byte of the data.
