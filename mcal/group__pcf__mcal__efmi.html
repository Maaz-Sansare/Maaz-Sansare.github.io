◆ EnEfmiPSize_t

enum EnEfmiPSize_t


Enumeration to define the parallelism size of Programming(write) and erasing the flash It represents the number of bytes to be programmed or erased when a write or erase operation occurs to the Flash memory.

Note
The parallelism size supported will depend on the microcontroller. The supported psize may also depend on the voltage supplied. Please refer to the microcontrollers datasheet for more details.
Enumerator
enEfmiPsize_Byte 

Erase and/or Program will happen 1 byte at a time

enEfmiPsize_HalfWord 

Erase and/or Program will happen 2 bytes at a time

enEfmiPsize_Word 

Erase and/or Program will happen 4 bytes at a time

enEfmiPsize_DoubleWord 

Erase and/or Program will happen 8 bytes at a time

Write/program data to the address.

The data is written using the parallelism represented by psize.

Parameters

data[IN] - Pointer to data to be programmed
size[IN] - The size, in bytes, of data to be programmed
psize[IN] - The parallelism size. It represents the number of bytes to be written in a single write operation to the Flash memory
verifyWrite[IN] - If enPCFoolean_True verify the data write was successful.
Returns
pcf_status_t Valid values :
  • PCF_EFMI_S_SUCCESS : Successful operation.
  • PCF_EFMI_S_BUSY : Memory operation is in progress.
  • PCF_EFMI_S_INVALID_OPERATION : Specified sector is write protected.
