Detailed Description

Can configuration structure.

Example to configure CAN:

CanHWConfig_t hw_config = {0};
hw_config.tx.port = PCF_PORT_0;
hw_config.tx.pin = PCF_PIN_1;
hw_config.rx.port = PCF_PORT_0;
hw_config.rx.pin = PCF_PIN_0;
CanConfig_t config = {0};
config.id = PCF_CAN4_ID;
config.baudrate = 500000;
config.txBufferSize = 16; // Tx buffer size 16 (Non-Blocking mode)
config.rxBufferSize = 16; // Rx buffer size 16 (Non-Blocking mode)
config.handleErrors = enPCFBoolean_True;
config.disableAutoReTx = enPCFBoolean_True;
config.ErrorHandler = __errorHandler;
config.hardwareCfg = &hw_config;

◆ handleErrors

EnPCFBoolean_t CanConfig_t::handleErrors

enPCFBoolean_True or enPCFBoolean_False. If enPCFBoolean_False the driver will not handle Transmit warning, error and Bus off states. It is expected the same will be handled by the user.

◆ disableAutoReTx

EnPCFBoolean_t CanConfig_t::disableAutoReTx

enPCFBoolean_True or enPCFBoolean_False. If enPCFBoolean_True the driver will not auto re-transmit the message on error. It is expected the same will be handled by the user.
