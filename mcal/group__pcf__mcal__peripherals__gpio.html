Detailed Description

General-Purpose Input Output (GPIO) is a digital pin of an IC.

It can be used as input or output for interfacing devices.

Note
Input validation is done only when package is built in debug mode.

◆ EnGpioOutputType_t

enum EnGpioOutputType_t

Defines the type of Output: Push-Pull or Open Drain.

Push-Pull outputs can pull a pin to either ‘1’ or ‘0’ while Open Drain outputs can only pull the pin to ‘0’ In Open Drain outputs to achieve a logical high output on the line, a pull-up resistor is used to connect the Open-Drain output to the desired output voltage level.

Note
This configuration is applicable only when GPIO direction is output.

◆ pcf_Gpio_new()

PCF_MCAL_API Gpio_t * pcf_Gpio_new ( const GpioConfig_t *const  config,
pcf_status_t *const  status 
)

Creates a new instance of Gpio_t based on the configuration provided by GpioConfig_t config .

Parameters
config[IN] - Pointer to the gpio configuration structure.
status[OUT] - Pointer to pcf_status_t (Optional parameter, pass PCF_NULL if not required.) Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : gpio is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid port,pin,direction,output type or speed was passed.(Checked only in debug mode)
  • PCF_S_ALREADY_IN_USE : Specified port pin is already in use.
  • PCF_S_NOT_ENOUGH_MEMORY :Not enough memory to create the Gpio entity.
Returns
valid values:
  • On Success: A pointer to the newly created Gpio_t object.
  • On Failure: PCF_NULL : The cause of the error can be identified by examining the value of the status variable.
