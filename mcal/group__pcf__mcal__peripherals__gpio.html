◆ EnGpioOutputType_t

enum EnGpioOutputType_t

GPIO Output Type.

Defines the type of Output: Push-Pull or Open Drain. Push-Pull outputs can pull a pin to either ‘1’ or ‘0’ while Open Drain outputs can only pull the pin to ‘0’ In Open Drain outputs to achieve a logical high output on the line, a pull-up resistor is used to connect the Open-Drain output to the desired output voltage level.

Note
This configuration is applicable only when GPIO direction is output.
Enumerator
enGpioOutputType_PushPull 

Gpio Output Push-Pull with no pull-up or pull-down

◆ pcf_Gpio_new()

PCF_MCAL_API Gpio_t * pcf_Gpio_new ( const GpioConfig_t *const  config,
pcf_status_t *  status 
)

Creates a new Gpio Entity Object.

It creates and initializes Gpio Entity according to the configuration structure GpioConfig_t and returns a pointer to Gpio_t.

Parameters
config[IN] - Pointer to the gpio configuration structure.
status[OUT] -
  • PCF_GPIO_S_SUCCESS : Successful operation.
  • PCF_GPIO_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_GPIO_S_INVALID_ARG : Invalid port,pin,direction,output type or speed was passed.(Checked only in debug mode)
  • PCF_GPIO_S_ALREADY_IN_USE : Specified port pin is already in use.
  • PCF_GPIO_S_ALLOCATION_FAILED : Memory allocation failed.
Returns
Pointer of the Gpio_t object or PCF_NULL in case of error.
