Detailed Description

A watchdog timer is a hardware timer that automatically generates a system reset if the main program neglects to periodically service it.

It is often used to automatically reset an embedded device that hangs because of a software or hardware fault.

Note
Input validation is done only when package is built in debug mode.

◆ pcf_Wdt_init()

PCF_MCAL_API pcf_status_t pcf_Wdt_init ( const WdtConfig_t *const  config)

Initialize the watchdog timer.

Note
Other watchdog timer API functions will return error if init is not called.
Watchdog timer is a singleton. Hence, there is no constructor (new) and destructor (delete).
Parameters
config[IN] - Pointer to the WdtConfig_t configuration structure.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : config or config members is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : config.timeout value or config.timeout unit is invalid.
