◆ pcf_Wdt_init()

PCF_MCAL_API pcf_status_t pcf_Wdt_init ( const WdtConfig_t *  config)

Initialize the watchdog timer.

Other wdt API functions will return error if init is not called.

Note
Watchdog timer is a singleton. Hence, there is no constructor (new) and destructor (delete).
Parameters
config[IN] - Pointer to the wdt configuration structure.
Returns
pcf_status_t Valid values :
  • PCF_WDT_S_SUCCESS : Successful operation.
  • PCF_WDT_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_WDT_S_INVALID_ARG : Timeout value or timeout unit was invalid.

◆ pcf_Wdt_start()

PCF_MCAL_API pcf_status_t pcf_Wdt_start ( )

Start watchdog timer.

After the timeout specified during initialization a wdt timeout event will occur and the microcontroller will be reset.

Returns
pcf_status_t Valid values :
  • PCF_WDT_S_SUCCESS : Successful operation.

◆ pcf_Wdt_stop()

PCF_MCAL_API pcf_status_t pcf_Wdt_stop ( )

Stop watchdog timer.

It may be started again by a call to the function pcf_Wdt_start.

Note
This function is an optional function. It will be implemented if the platform supports the same.
Returns
pcf_status_t Valid values :
  • PCF_WDT_S_SUCCESS : Successful operation.
