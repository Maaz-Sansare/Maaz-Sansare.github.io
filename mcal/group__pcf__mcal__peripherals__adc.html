Detailed Description

ADC refers to analog to digital converter which is a device that converts continuously changing analog signals into discrete digital signals.

Analog signals, such as temperature, pressure, sound, or images, need to be converted into digital forms that are easier to be stored, processed, and transmitted.

Note
Input validation is done only when package is built in debug mode.

◆ pcf_Adc_addChannel()

PCF_MCAL_API pcf_status_t pcf_Adc_addChannel ( Adc_t *const  adc,
const uint8_t  channel_id,
const EnAdcGroupType_t  group,
const uint32_t  conv_cycles 
)

Adds a channel into a ADC group.

If a particular channel is added to a group (regular or injected) it cannot be added to another group.

Parameters
adc[IN] - Pointer to Adc_t entity object.
channel_id[IN] - Channel id.
group[IN] - The group to which the channel should be added. Regular or Injected.
conv_cycles[IN] - The conversion cycles in number of clock cycles depending on the frequency set during configuration. conversion cycles (conv_cycles) = conversion time * Adc frequency eg adc freq = 14MHz, conversion time = 1us hence conversion cycles = 14Mhz * 1us = 14.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : adc is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : adc or channel_id (or group (based on uC)) is invalid. (Checked only in debug mode)
  • PCF_S_ALREADY_IN_USE : Specified channel_id is already in use.
