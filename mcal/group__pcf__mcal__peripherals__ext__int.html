Enumeration Type Documentation

◆ EnExtIntTrigger_t

enum EnExtIntTrigger_t

Enumeration for external interrupt trigger selection What will cause the interrupt to be triggered.

Note: All options may not be applicable to a microcontroller/platform. Refer microcontroller/platform documentation/datasheet/reference manuals to know which triggers are applicable.

Enumerator
enExtIntTrigger_FallingEdge 

Falling edge triggered interrupt.


Interrupt will be triggered when signal on the pin transitions from high to low

enExtIntTrigger_RisingEdge 

Rising edge triggered interrupt.

Interrupt will be triggered when signal on the pin transitions from high to low

enExtIntTrigger_BothEdge 

Interrupt on both edge.

Interrupt will be triggered when signal on the pin transitions from high to low and from low to high

enExtIntTrigger_LowLevel 

Low level triggered interrupt.

Interrupt is triggered if Pin state is OFF/Low

enExtIntTrigger_HighLevel 

High level triggered interrupt.

Interrupt is triggered if Pin state is ON/High

enExtIntTrigger_BothLevel 

Interrupt on both level.

Interrupt is triggered if Pin state is either On or Off

enExtIntTrigger_BoundSize 

To ensure sizeof(EnExtIntTrigger_t) == 4 bytes

Creates a new instance of the ExtInt_t structure based on the configuration structure ExtIntConfig_t and returns a pointer to ExtInt_t.

◆ pcf_ExtInt_attachCallback()

PCF_MCAL_API pcf_status_t pcf_ExtInt_attachCallback ( const ExtInt_t *const  ext_int,
const ExtIntCallback_fp  callback,
const pcf_generic_t  user_args,
ExtIntCallback_fp *const  old_callback,
pcf_generic_t *const  old_userArgs 
)

Attaches a callback to the external interrupt.

This callback will be called whenever the interrupt is fired.

Note
Call this function before enabling the external interrupt device. To attach a callback if external interrupt has already been enabled do the following
  1. Disable external interrupt pcf_ExtInt_disable()
  2. Attach a callback pcf_ExtInt_attachCallback()
  3. Enable external interrupt device again pcf_ExtInt_enable()
Parameters
ext_int[IN] - Pointer to ext_int entity object.
callback[IN] - Pointer to the callback function to be attached.
user_args[IN] - User-defined arguments to be passed to the callback function.
old_callback[OUT] - The previous callback registered with the external interrupt. Optional parameter, pass PCF_NULL if you do not want the callback previously registered.
old_userArgs[OUT] - The previous user arguments registered with the external interrupt. Optional parameter, pass PCF_NULL if you do not want the older user defined data.
Returns
pcf_status_t Valid values:
  • PCF_EXTI_S_SUCCESS : Successful operation.
  • PCF_EXTI_S_NULL_ARG : Null ext_int pointer is passed. (Checked only in debug mode)

◆ pcf_ExtInt_setPriority()

PCF_MCAL_API pcf_status_t pcf_ExtInt_setPriority ( const ExtInt_t *const  ext_int,
const uint32_t  priority,
const uint32_t  sub_priority 
)

Sets the global interrupt priority, of the external interrupt, across all entities which are interrupt based.

Note
lower the priority value, higher the priority.
Parameters
ext_int[IN] - Pointer to ext_int entity object
priority[IN] - The priority level, lower priority value means higher priority
sub_priority[IN] - The sub priority level, This fields validity is dependant on value of

PCF_CONF_CPU_IC_PRIORITY_GROUP configured in cpu package, if two interrupts have the same priorities then the interrupt with higher sub-priority will be triggered first.

Returns
pcf_status_t Valid values:
  • PCF_EXTI_S_SUCCESS : Successful operation.
  • PCF_EXTI_S_NULL_ARG : Null ext_int pointer is passed. (Checked only in debug mode)
