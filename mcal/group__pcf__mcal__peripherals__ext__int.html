Detailed Description

The External Interrupt interface is used to trigger an event when a digital input pin changes.

You can trigger interrupts on the rising edge (change from 0 to 1) or falling edge (change from 1 to 0) of signals or both.

External interrupts are hardware-triggered events that allow a microcontroller or microprocessor to respond to external signals or events, providing a mechanism for real-time event-driven processing in embedded systems. These interrupts enable the processor to momentarily suspend its current task and execute a specific interrupt service routine, making them crucial for handling time-sensitive events in embedded applications.

Note
Input validation is done only when package is built in debug mode.

Enumeration Type Documentation

◆ EnExtIntTrigger_t

enum EnExtIntTrigger_t

Enumeration for external interrupt trigger mode selection, i.e.

what will cause the interrupt to be triggered.

Note: All options may not be applicable to a microcontroller/platform. Refer microcontroller/platform documentation/datasheet/reference manuals to know which triggers are applicable.

Enumerator
enExtIntTrigger_FallingEdge 

Falling edge triggered interrupt.
Interrupt will be triggered when signal on the pin transitions from high to low

enExtIntTrigger_RisingEdge 

Rising edge triggered interrupt. Interrupt will be triggered when signal on the pin transitions from high to low

enExtIntTrigger_BothEdge 

Interrupt on both edge. Interrupt will be triggered when signal on the pin transitions from high to low and from low to high

enExtIntTrigger_LowLevel 

Low level triggered interrupt. Interrupt is triggered if Pin state is OFF/Low

enExtIntTrigger_HighLevel 

High level triggered interrupt. Interrupt is triggered if Pin state is ON/High

enExtIntTrigger_BothLevel 

Interrupt on both level. Interrupt is triggered if Pin state is either On or Off

◆ pcf_ExtInt_attachCallback()

PCF_MCAL_API pcf_status_t pcf_ExtInt_attachCallback ( ExtInt_t *const  ext_int,
const ExtIntCallback_fp  callback,
const pcf_generic_t  user_args,
ExtIntCallback_fp *const  old_callback,
pcf_generic_t *const  old_userArgs 
)

Attaches/Detaches a callback to the external interrupt.

This callback will be called whenever the interrupt is fired.

Note
if callback & user_arg is PCF_NULL then api will detach the previously attached callback.
Call this function before enabling the external interrupt device. To attach a callback if external interrupt has already been enabled do the following
  1. Disable external interrupt using pcf_ExtInt_enable
  2. Attach a callback pcf_ExtInt_attachCallback
  3. Enable external interrupt using pcf_ExtInt_enable
Parameters
ext_int[IN] - Pointer to ExtInt_t entity object.
callback[IN] - Pointer to ExtIntCallback_fp, callback function to be attached.
user_args[IN] - User-defined arguments to be passed to the callback function.
old_callback[OUT] - Pointer to ExtIntCallback_fp, previous callback registered with the external interrupt.
(Optional parameter, pass PCF_NULL if you do not want the previous callback registered.)
old_userArgs[OUT] - The previous user arguments registered with the external interrupt.
(Optional parameter, pass PCF_NULL if you do not want the previous value registered.)
Returns
pcf_status_t Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : ext_int or callback is PCF_NULL. (Checked only in debug mode)

PCF_MCAL_API pcf_status_t pcf_ExtInt_setPriority ( ExtInt_t *const  ext_int,
const uint32_t  priority,
const uint32_t  sub_priority 
)

Sets the global interrupt priority, of the external interrupt.

The priority is set across all entities which are interrupt based.

Note
lower the priority value, higher the priority.
Parameters
ext_int[IN] - Pointer to ExtInt_t entity object
priority[IN] - The priority level, lower priority value means higher priority
sub_priority[IN] - The sub priority level, This fields validity is dependant on value of PCF_CONF_CPU_IC_PRIORITY_GROUP configured in mcal package, if two interrupts have the same priorities then the interrupt with higher sub-priority will be triggered first.
Returns
pcf_status_t Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : ext_int is PCF_NULL. (Checked only in debug mode)
