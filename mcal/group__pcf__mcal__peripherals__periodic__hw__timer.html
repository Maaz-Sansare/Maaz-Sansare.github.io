Detailed Description

A periodic hardware timer is a specialized component that generates regular,
precisely-timed interrupts or events at fixed intervals.

It is commonly used for tasks requiring precise timing, such as scheduling tasks, measuring time intervals, or controlling periodic operations in applications like real-time systems and data logging.

◆ pcf_PeriodicHWTimer_attachCallback()

PCF_MCAL_API pcf_status_t pcf_PeriodicHWTimer_attachCallback ( PeriodicHWTimer_t *const  p_timer,
const PeriodicTimerCallback_fp  callback,
const pcf_generic_t  user_arg,
PeriodicTimerCallback_fp *const  old_callback,
pcf_generic_t *const  old_userArg 
)

Attaches/Detaches a callback, to be called at a recurring interval with which the periodic timer was configured.

Note
if callback & user_arg is PCF_NULL then api will detach the previously attached callback.
Call this function before enabling the periodic HW timer device. To attach/detaches a callback if periodic HW timer has already been enabled do the following
  1. Disable periodic HW timer using pcf_PeriodicHWTimer_start
  2. Attach a callback pcf_PeriodicHWTimer_attachCallback
  3. Enable periodic HW timer using pcf_PeriodicHWTimer_start
Parameters
p_timer[IN] - Pointer to PeriodicHWTimer_t entity object.
callback[IN] - User callback to be called at the specified compare_time
user_arg[IN] - User data that will be passed to the callback. Can be set to NULL.
old_callback[OUT] - The callback previously registered. Will be set to NULL if no callback was registered (Optional parameter, pass PCF_NULL if you do not want the callback previously registered.)
old_userArg[OUT] - The previous user data set. (Optional parameter, pass PCF_NULL if you do not want the older user defined data.)
Returns
pcf_status_t Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : p_timer is PCF_NULL. (Checked only in debug mode)
