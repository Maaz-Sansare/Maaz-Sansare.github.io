Detailed Description

A GPIO bank is a group or set of GPIO (General-Purpose Input/Output) pins on a microcontroller that are often grouped together and share common characteristics This grouping allows for easier management and configuration of multiple GPIO pins at once, making it more convenient to work with hardware components that require similar settings or functions.

Note
Input validation is done only when package is built in debug mode.

◆ pcf_GpioBank_new()

PCF_MCAL_API GpioBank_t * pcf_GpioBank_new ( const GpioBankConfig_t *const  config,
pcf_status_t *  status 
)

Creates a new instance of GpioBank_t based on the configuration provided by GpioBankConfig_t config .

Parameters
config[IN] - Pointer to the gpio bank configuration structure.
status[OUT] - Pointer to pcf_status_t (Optional parameter, pass PCF_NULL if not required.) Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : gpio_bank is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid port,pin,direction,output type or speed was passed.(Checked only in debug mode)
  • PCF_S_ALREADY_IN_USE : Specified port pin is already in use.
  • PCF_S_NOT_ENOUGH_MEMORY : Not enough memory to create the Gpio bank entity.
Returns
valid values:
  • On Success: A pointer to the newly created GpioBank_t object.
  • On Failure: PCF_NULL : The cause of the error can be identified by examining the value of the status variable.

◆ pcf_GpioBank_write()

PCF_MCAL_API pcf_status_t pcf_GpioBank_write ( const GpioBank_t *const  gpio_bank,
const uint32_t  value 
)

Write value to the gpio_bank .

(This can only be used when gpio bank direction is set to Output).

Parameters
gpio_bank[IN] - Pointer to GpioBank_t object.
value[IN] - An integer where each bit of the integer corresponds to an associated port pin. If bit is high the pin is set if bit is low the pin is cleared. The LSb is mapped to the starting pin of the Bank. bit 0 of value -> First Pin in the Bank bit 1 of value -> Second Pin in the Bank bit N of value -> Nth Pin in the Bank.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : gpio_bank is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_OPERATION : Gpio direction is not in output mode.

◆ oType

EnGpioOutputType_t GpioBankConfig_t::oType

Gpio Bank Pins output type. Push-Pull or Open-Drain. The value can be set even if direction is Input, However the configuration comes into effect only when direction is Output If direction is Input and this value is not set then by default Output Type is set as Push-Pull

◆ speed

EnGpioSpeed_t GpioBankConfig_t::speed

Gpio Bank Pins transition speed/slew rate. Time taken to go from one state to another. The value can be set even if direction is Input, However the configuration comes into effect only when direction is Output If direction is Input and this value is not set then by default Speed is set as Low
