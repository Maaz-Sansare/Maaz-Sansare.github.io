Detailed Description

UART refers to Universal Asynchronous Receiver/Transmitter, as a kind of asynchronous serial communication protocol, the working principle is to transmit each character of the transmitted data one by one.

Note
Input validation is done only when package is built in debug mode.

Creates a new instance of Uart_t based on the configuration provided by UartConfig_t config .

  • PCF_S_NOT_ENOUGH_MEMORY : Not enough memory to create the Uart entity.
Returns
valid values:

◆ pcf_Uart_setPriority()

PCF_MCAL_API pcf_status_t pcf_Uart_setPriority ( const Uart_t *const  uart,
const interrupt_t  tx_priority,
const interrupt_t  tx_sub_priority,
const interrupt_t  rx_priority,
const interrupt_t  rx_sub_priority 
)

Sets the global interrupt priority, of the Tx and Rx for Uart_t object depending on mode of uart's Tx and Rx.

Note
uart Tx can be either in DMA,INT or POL mode.
uart Rx can be either in DMA,INT or POL mode.
The mode of uart is decided via config at time of creation of Uart_t object using pcf_Uart_new.
Parameters
uart[IN] - A pointer to Uart_t object.
tx_priority[IN] - NVIC priority level for uart Tx, lower value indicates higher priority.
tx_sub_priority[IN] - NVIC sub-priority level for uart Tx, lower value indicates higher sub-priority.
rx_priority[IN] - NVIC priority level for uart Rx, lower value indicates higher priority.
rx_sub_priority[IN] - NVIC sub-priority level for uart Rx, lower value indicates higher sub-priority.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Uart_t object is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Uart_t is invalid.(Checked only in debug mode)
Note
If uart Tx and Rx both are in INT mode, then only rx_priority and rx_sub_priority are taken into consideration while setting the priority.

◆ pcf_Uart_read()

PCF_MCAL_API pcf_status_t pcf_Uart_read ( const Uart_t *const  uart,
Buffer_t *const  buffer,
pcf_size_t *const  actual_size 
)

Receive/Read data on uart.

Run in polling (blocking) mode if rxBufferSize is set to 0 or system could allocate memory for rxBuffer. Run in DMA or interrupt (Non-Blocking) mode if rxBufferSize is set to any nonzero positive value.

The user provides a pointer to Buffer_t which stores the buffer address and size to be read in bytes. The function does not guarantee reading the size specified by the user. The actual number of bytes read is returned to user in variable actual_size.

Note
User can call pcf_Uart_getRxDataCount function to check if data is present in receive buffer or not.
Parameters
uart[IN] - Pointer to Uart_t object.
buffer[IN/OUT] - Pointer to Buffer_t, stores the data into the buffer of specified length.
actual[OUT] - Pointer to pcf_size_t, stores actual number of bytes read.
(Optional parameter, pass PCF_NULL if not required.)
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Uart_t object is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Uart_t is invalid.(Checked only in debug mode)

◆ pcf_Uart_write()

PCF_MCAL_API pcf_status_t pcf_Uart_write ( const Uart_t *const  uart,
const Buffer_t *const  buffer,
pcf_size_t *const  actual_size 
)

Send/Write data over uart.

Run in polling (blocking) mode if txBufferSize is set to 0 or system could allocate memory for txBuffer. Run in DMA or interrupt (Non-Blocking) mode if txBufferSize is set to any nonzero positive value.

Note
In Polling/Blocking mode, This function blocks until all bytes specified by user (size) are sent or return busy status.
In non-blocking mode, this function does not block for the actual transmission on the wire. It does not guarantee the transmission of all bytes specified by the user (size). It depends on the free space available in the internal buffer. The actual number of bytes sent is saved in the actual_size variable.
Parameters
uart[IN] - A pointer to Uart_t object.
buffer[IN] - Pointer to Buffer_t, writes the buffer of specified length to the uart.
actual[OUT] - Pointer to pcf_size_t, stores actual number of bytes sent successfully. (Optional parameter, pass PCF_NULL if not required.)
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : Uart_t object is PCF_NULL.(Checked only in debug mode)
  • PCF_S_INVALID_ARG : Uart_t is invalid.(Checked only in debug mode)
