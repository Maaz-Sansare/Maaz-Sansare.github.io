◆ pcf_Uart_setPriority()

PCF_MCAL_API pcf_status_t pcf_Uart_setPriority ( const Uart_t *const  uart,
const uint32_t  tx_priority,
const uint32_t  tx_sub_priority,
const uint32_t  rx_priority,
const uint32_t  rx_sub_priority 
)

Set the Tx and Rx interrupt priority of a particular Uart_t object depending on mode of uart's Tx and Rx.

Note
uart Tx can be either in DMA,INT or POL mode.
uart Rx can be either in DMA,INT or POL mode.
The mode of uart is decided via config at time of creation of Uart_t object using pcf_Uart_new.
Parameters
uart[IN] - Pointer to Uart_t.
tx_priority[IN] - NVIC priority level for uart Tx, lower value indicates higher priority
tx_sub_priority[IN] - NVIC sub-priority level for uart Tx, lower value indicates higher sub-priority
rx_priority[IN] - NVIC priority level for uart Rx, lower value indicates higher priority
rx_sub_priority[IN] - NVIC sub-priority level for uart Rx, lower value indicates higher sub-priority
Returns
pcf_status_t Valid values :
  • PCF_UART_S_SUCCESS : Successful operation.
  • PCF_UART_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_UART_S_INVALID_ARG : Invalid Uart_t object was passed.(Checked only in debug mode)
Note
If uart Tx and Rx both are in INT mode, then only "rx_priority" and "rx_sub_priority" are taken into consideration while setting the priority.

◆ pcf_Uart_read()

PCF_MCAL_API pcf_status_t pcf_Uart_read ( const Uart_t *const  uart,
pcf_byte_t *  data,
const pcf_size_t  size,
pcf_size_t *  actual_size 
)

Receive/Read data on uart.

If the system could not allocate memory for the RxBuffer during creation then the function is blocking else it is non-blocking. The user provides a pointer to a data buffer and size (the number of bytes) that need to be read. The function does not guarantee reading the size specified by the user. The actual number of bytes read is returned to user in variable actual_size.

User may Call pcf_Uart_getRxBufferCount() function to check if data is present in receive buffer or not. Call this function to read the buffer.

Parameters
uart[IN] - Pointer to Uart_t
data[OUT] - Pointer to data buffer where data will be copied.
size[IN] - Total number of bytes user wants to read
actual_size[OUT] - Total number of bytes read successfully.
Returns
pcf_status_t Valid values :
  • PCF_UART_S_SUCCESS : Successful operation.
  • PCF_UART_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_UART_S_INVALID_ARG : Invalid Uart_t object was passed.(Checked only in debug mode)

◆ pcf_Uart_write()

PCF_MCAL_API pcf_status_t pcf_Uart_write ( const Uart_t *const  uart,
const pcf_byte_t *  data,
const pcf_size_t  size,
pcf_size_t *  actual_size 
)

Send/Write data over uart.

Run in polling (Blocking) mode if txBufferSize is set to 0. Run in DMA or interrupt (Non-Blocking) mode if txBufferSize is set to any nonzero positive value.

In Polling/Blocking mode This function blocks until all bytes specified by user (size) are sent on the wire. The function will guarantee transmission of all bytes.

In Non-Blocking mode This function will not block for actual transmission on the wire. The function will not guarantee transmission of all bytes specified by user (size). It depends on free space available in internal buffer. The actual number of bytes sent is saved in variable actual_size.

Parameters
uart[IN] - Pointer to Uart_t.
data[IN] - Pointer to data buffer that needs to be transmitted
size[IN] - Total number of bytes to be sent.
actual_size[OUT] - Total number of bytes sent successfully.
Returns
pcf_status_t Valid values :
  • PCF_UART_S_SUCCESS : Successful operation.
  • PCF_UART_S_NULL_ARG : Null argument was passed.(Checked only in debug mode)
  • PCF_UART_S_INVALID_ARG : Invalid Uart_t object was passed.(Checked only in debug mode)
