Function macros
MACRO DESCRIPTION
__PCF_FUNC__ Holds the name of the current function as a string.
__PCF_LINE__ Holds the current input line number, in the form of a decimal integer constant.
__PCF_FILE__ Holds the name of the current input file.
__PCF_DECORATED_FUNCTION__ Redefined identifiers that contain the name of the lexically-enclosing function.
PCF_FORCE_INLINE Indicates that a function must be inlined
PCF_NO_INLINE Prevents the compiler from making any use of a constant data value for optimization purposes, without affecting its placement in the object
PCF_INLINE Instruct the compiler to insert a copy of the function body into each place the function is called
PCF_STATIC_INLINE The inline function which can be used in this file only.
PCF_NO_RETURN Informs the compiler that the function does not return
PCF_ASM Invokes the inline assembler and can appear wherever a C or C++ statement is legal
PCF_DEF_PACKED_STRUCT Remove padding between members of struct
PCF_DEF_ALIGNED_VAR Specifies a minimum alignment for the function, measured in bytes
PCF_EXPORT_API Export directive to the object file
PCF_IMPORT_API Import functions, data, and objects to a DLL
PCF_LOCAL_API No other module (executable or shared library) can reference it directly.
PCF_DEPRECATED Results in a warning if the function is used anywhere in the source file
PCF_STATIC_ASSERT Performs compile-time assertion checking
PCF_BEGIN_EXTERN Start declaration in header file for building library in C.
PCF_END_EXTERN End declaration in header file for building library in C.
PCF_NULL A single character that compares equal to 0.
PCF_UNUSED Suppresses unused variable compiler warning.
PCF_BUILD_TIME Holds the compilation time and date as a string.
PCF_FILE_MODIFICATION_TIMESTAMP Holds the file modification time,date and year as a string.
PCF_COUNTER Holds sequential integral values starting from 0. In conjunction with the ## operator, this provides a convenient means to generate unique identifiers.
PCF_CDECL_API Causes the compiler to assume that the calling function will pop off the stack space used to pass arguments.
PCF_STDCALL_API Causes the compiler to assume that the called function will pop off the stack space used to pass arguments, unless it takes a variable number of arguments.
PCF_MACRO_BEGIN Beginning of construct that is use to define a multistatement operation.
PCF_MACRO_END End of construct that is use to define a multistatement operation.
PCF_COLD_FUNC_ATTR Informs the compiler that a function is unlikely executed.
PCF_BUILTIN_WCHAR Defined if wchar_t type is present.
PCF_WCHAR_MAX Defined to the maximum value of the wchar_t type.
PCF_WCHAR_MIN Defined to the minimum value of the wchar_t type.
PCF_WCHAR_WIDTH Defined to the bit widths of the wchar_t types.
PCF_NOOP Instruction to do nothing.
PCF_UNUSED_FUNC When attached to a variable, means that the variable is meant to be possibly unused.
PCF_WARNING_PUSH Saves diagnostic state.
PCF_WARNING_POP Restore diagnostic state to the last PCF_WARNING_PUSH
PCF_WARNING_DISABLE Ignores the diagnostic.
PCF_WARNING_SUPPRESS Suppresses any warning by the diagnostic.
PCF_HAS_WARNING Checks for the given warning.
PCF_UNREACHABLE Indicates the compiler optimizer that a certain section of the code would never be reachable.
PCF_ASSUME Informs the compiler that a particular condition can be assumed to be true.
PCF_LIKELY Informs the compiler that the given code is most probably executed.
PCF_UNLIKELY Informs the compiler that the given code is least probably executed.
PCF_PRINTF_FORMAT_ATTR Specifies that a function takes printf style arguments which should be type-checked against a format string.
PCF_ALIGN_OF Returns the alignment of the given type, in bytes
PCF_DEBUG Sets in debug mode.
