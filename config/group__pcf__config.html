Detailed Description

Configuration package to be used by other PCF C packages and applications using PCF C packages.

This package sets up the build configurations and common types depending on the platform, i.e. architecture, OS, language and compiler detected.

Note
This is a required package for all PCF C packages and applications using PCF C packages. For using the @pcf/config package you should include “pcf/config/config.h” in your package and/or application.
Macros which start with "__" or "_" and doesn't end with "__" are reserved to support internal infrastructure and not meant to be used in your code.

)     PCF_CONCAT_4(PATH, PACKAGE, MODULE, INCLUDE_NAME)

Concatenates the arguments passed to create a file path that conforms to the path format used in C for include files.

Used for defining the file to be included.

For example including "item_one.h" from "my_package" under scope "my_team":

#define CONDITIONAL_INCLUDE PCF_CONDITIONAL_INCLUDE_FILE(my_team/,my_package/,item_one/,item_one.h)
#include CONDITIONAL_INCLUDE
#undef CONDITIONAL_INCLUDE

CONDITIONAL_INCLUDE = my_team/my_package/item_one/item_one.h

Parameters
PATH- Directory of the project.
PACKAGE- Package name.
MODULE- Package_Item name.
INCLUDE_NAME- File name.
Returns
Concatenated file path.
