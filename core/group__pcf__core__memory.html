Detailed Description

The "memory" entity provide memory management functions.

It allows user to allocate and manipulate memory. It provides functions such as memory allocation, deallocation, copying, comparison, etc.

Note
Input validation is done only when package is built in debug mode.

◆ pcf_Memory_alloc()

PCF_CORE_API pcf_generic_t pcf_Memory_alloc ( const pcf_size_t  size,
pcf_status_t *  status 
)

Dynamically allocate the memory and initializes it to 0.

Parameters
size[IN] - Size of the memory to be allocated of type pcf_size_t.
status[OUT] - Pointer to pcf_status_t, stores the status of the function call. (Optional parameter, pass PCF_NULL if not required.) Valid values:
  • PCF_S_SUCCESS : On Success.
  • PCF_S_INVALID_ARG : size is 0. (Checked only in debug mode)
  • PCF_S_NOT_ENOUGH_MEMORY : Failed to allocate specified size of memory.
Returns
  • On success - The pointer to the allocated memory.
  • On Failure - PCF_NULL.
Examples
array_example.c, entity_manager_example.c, linear_buffer_example.c, and ring_buffer_example.c.

  • PCF_S_SUCCESS : On Success.
  • PCF_S_NULL_ARG : dest or src is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : Is due to memory range overlap or size is 0. (Checked only in debug mode)
Examples
array_example.c.

◆ pcf_Memory_move()

PCF_CORE_API pcf_status_t pcf_Memory_move ( pcf_generic_t  dest,
pcf_generic_t  src,
const pcf_size_t  size 
)

Same as pcf_Memory_copy and also allows memory overlap.

Copies size bytes from the location pointed by src to the memory block pointed by dest .

Note
The copy is not done directly the data from src is copied into a temporary buffer and then copied into dest . Hence, even if src and dest overlap there will be no problem.
Parameters
dest[IN/OUT] - Pointer to the destination memory where the content is to be copied.
src[IN] - Pointer to the source memory from where data is to be copied.
size[IN] - Number of bytes to copy of type pcf_size_t.
Returns
pcf_status_t Valid values :
  • PCF_S_SUCCESS : On Success.
  • PCF_S_NULL_ARG : dest or src is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : size is 0. (Checked only in debug mode)
  • PCF_S_NOT_ENOUGH_MEMORY : Failed to allocate memory.
Examples
array_example.c.

◆ pcf_Memory_chr()

PCF_CORE_API pcf_generic_t pcf_Memory_chr ( const pcf_generic_t  memory,
const pcf_byte_t  value,
const pcf_size_t  size,
pcf_status_t *  status 
)

Scans the initial size bytes of the memory area pointed to by memory for the first instance of value .

Parameters
memory[IN] - Pointer to block of memory.
value[IN] - Character value.
size[IN] - Number of bytes to analyzed.
status[OUT] - Pointer to pcf_status_t, stores the status of the function call.
(Optional parameter, pass PCF_NULL if not required.)
Valid values:
  • PCF_S_SUCCESS : On Success.
  • PCF_S_NULL_ARG : memory is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : size is 0. (Checked only in debug mode)
  • PCF_S_DATA_NOT_FOUND : value not found.
Returns
  • On success - The pointer to the matching byte.
  • On Failure - PCF_NULL if the character does not occur.
