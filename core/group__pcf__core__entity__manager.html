Detailed Description

The "entity manager" maintains a collection of entities.

The manager provides functionalities to add new entities, remove entities and various other operations.

Note
Input validation is done only when package is built in debug mode.

◆ pcf_EntityManager_init()

PCF_CORE_API pcf_status_t pcf_EntityManager_init ( const EntityManagerConfig_t *const  config)

Initialises the entity manager.

Note
Optionally, the user can provide their own memory allocation, reallocation, and deallocation functions.
By default, the entity manager uses the pcf/memory APIs for memory operations.
Parameters
config[IN] - Pointer to EntityManagerConfig_t to set array configurations.
Returns
pcf_status_t Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : config is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : config.max_entities is 0. (Checked only in debug mode)
  • PCF_S_NOT_ENOUGH_MEMORY : Not enough memory for allocation.
  • PCF_S_ALREADY_CONFIGURED : Entity manager is already configured.
Examples

PCF_CORE_API pcf_status_t pcf_EntityManager_increaseSize ( const pcf_size_t  num_entities)

Increases the size of the Entity manager by the specified num_entities.


It reallocates memory for the entity list to accommodate the increased size.

Parameters
num_entities[IN] - The number of entities to increase the size by.
Returns
pcf_status_t[OUT] Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NOT_CONFIGURED : Entity manager is not configured (Checked only in debug mode)
  • PCF_S_INVALID_ARG : Invalid num_entities is passed (Checked only in debug mode)
  • PCF_S_NOT_ENOUGH_MEMORY : Memory allocation failure
