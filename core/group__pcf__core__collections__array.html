Detailed Description

The "Array" entity represents a dynamic array that stores a collection of elements.

It provides various functions for accessing and manipulating the elements in the array.

Note
Array is not thread-safe. If multiple threads access and modify the same Array instance concurrently, appropriate synchronization mechanisms should be implemented by the user to ensure thread safety.
Input validation is done only when package is built in debug mode.

◆ pcf_Array_new()

PCF_CORE_API Array_t * pcf_Array_new ( const ArrayConfig_t *const  config,
pcf_status_t *const  status 
)

Creates a new array entity object.

Note
Optionally, the user can provide their own memory allocation, reallocation, deallocation and memory copy functions.
To use default pcf/memory APIs set the array memory config to PCF_NULL.
Parameters
config[IN] - Pointer to ArrayConfig_t to set array configurations.
status[OUT] - Pointer to pcf_status_t, stores the status of the function call.
(Optional parameter, pass PCF_NULL if not required.)
Valid values:
  • PCF_S_SUCCESS : On success.
  • PCF_S_NULL_ARG : If config or config.memAlloc or config.memMove or config.memCopy or config.memfree is PCF_NULL. (Checked only in debug mode)
  • PCF_S_INVALID_ARG : If config.count is 0 (Checked only in debug mode), config.growthFactor or config.maxElementCount is invalid.
  • PCF_S_NOT_ENOUGH_MEMORY : Memory allocation failure.
Returns
  • On success - Pointer to Array_t.
  • On failure - PCF_NULL.
